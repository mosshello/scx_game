# 首批游戏详情页与百度统计技术规格书

## 目标
- 为首页轮播中的 6 款游戏新增可独立收录的静态详情页，并让首页入口优先进入详情页。
- 为百度统计预留实施事项；收到用户提供的站点统计代码后再接入，禁止虚构统计 ID。

## 影响范围
| 文件 | 原因 |
|---|---|
| `index.html` | 增加首页 canonical，不覆盖用户现有标题文案 |
| `js/app.js` | 建立 6 款游戏到详情页的路由映射 |
| `css/style.css` | 增加详情页共用样式 |
| `sitemap.xml` | 纳入 6 个详情页 URL |
| `games/*.html` | 6 个静态、可直接抓取的游戏详情页 |
| `.ai/project.md` | 记录项目结构与变更 |

## 实施步骤
1. 新建赌石刮刮乐、疯狂买买买、进化塔防、火车打僵尸、抓大鹅、落叶清理模拟器详情页。
2. 每页提供唯一 title、description、canonical、H1、静态正文、购买/领取状态、更新时间及面包屑结构化数据。
3. 首页轮播和对应游戏卡片先进入站内详情页，详情页再跳转店铺。
4. 更新 sitemap，并补充首页 canonical。
5. 待用户提供百度统计完整安装代码后，再追加到所有公开 HTML 页面。

## 验证方式
- `node --check js/app.js`
- 本地 HTTP 检查首页、6 个详情页与 sitemap 均返回 200。
- 检查页面 title、description、canonical、H1 和站内详情链接。
- 检查现有 `index.html` 用户改动未被覆盖。

## 回滚方案
- 删除新增的 `games/` 页面，并回退本规格书涉及的 `index.html`、`js/app.js`、`css/style.css`、`sitemap.xml` 对应改动。
